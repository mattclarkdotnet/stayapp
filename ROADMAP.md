# Stay Roadmap

## Priority Order

1. Basic scenarios are automated
- Create automated tests for each scenario in tests/SCENARIOS.md that use real apps, not stubs or fixtures.

2. Basic scenarios work across wake cycles
- With user help, test that basic scenarios continue to work correctly after sleep/wake cycles.


## Deferred Until Baseline Is Stable

1. Multiple workspaces
2. Apps with child windows
3. Full screen apps
4. Changes in monitor configuration between sleep and wake (default - do nothing)
5. Productization
- App icon and branding assets.
- Start-on-login hardening.
- Packaging/signing/notarization/distribution workflows.
